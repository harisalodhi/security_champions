Why Cybersecurity Matters

Increasing threats and cyberattacks

  • Cyberattacks are becoming more sophisticated, complex, and frequent.

Cost of cybersecurity breach:

  • Financial impact: The average cost of a data breach is around $4.88 million in 2024 (according to IBM Data Breach Report), a 10% increase from the year before.
  • Human impact: Exposure of private and sensitive data, damage to reputation and loss of customer trust

Human error

  • Contributes to 95% of all breaches (IBM Cyber Security Intelligence Index Report).

Role of Security Champions

  • Security Champions act as role models, spreading awareness and implementing key changes

The Human Element

  • Weak or reused passwords
  • Unsecured devices and networks
  • Neglecting software updates
  • Phishing emails: Fraudulent emails designed to trick users.
  • Social engineering: Manipulating people into divulging confidential information.

Recognising Threats and Vulnerabilities

Phishing Attacks

Red Flags:

  • Unusual sender address(e.g., “[email protected]” instead of a legitimate domain)
  • Unexpected links or attachments
  • Generic greetings like "Dear User" instead of personalized names

What to do:

  • Verify sender details
  • Report suspicious emails
  • Access linked sites from the web instead of the email hyperlink

Malware Delivery

Red Flags:

  • Unexpected file attachments
  • Pop-ups urging downloads
  • Slow device performance after opening unknown file

What to do:

  • Avoid opening attachments or files from unknown sources
  • Use antivirus software to scan downloads

Social Engineering

Red Flags:

  • Unexpected calls or messages claiming to be from authority figures (e.g., CEO or IT support)
  • Requests for sensitive data like passwords or financial details
  • Appeals to emotions like fear (e.g., threats of account deactivation) or urgency

What to do:

  • Verify the identity of the requester
  • Avoid sharing information over the phone or email

Best Practices for a Secure Culture

Use strong and unique passwords

  • Create passwords with a mix of uppercase, lowercase, numbers, and symbols (e.g., P@ssw0rd!23)
  • Use a password manager to generate and store complex passwords securely

Enable Multi-Factor Authentication (MFA)

  • Use MFA wherever possible (e.g., work accounts, email, and banking)
  • Choose apps like Google Authenticator or Microsoft Authenticator instead of SMS-based MFA for better security

Verify links and attachments before opening

  • Be cautious of unexpected attachments, even if they appear to come from a known contact
  • Access linked sites from the web instead of the email hyperlink

Practice secure device handling

  • Lock your screen when stepping away from your workstation
  • Avoid connecting to public Wi-Fi without a secure VPN

Report incidents immediately

  • Report suspicious emails, calls, or activities to the IT security team promptly
  • Encourage a “no-blame” culture to ensure employees feel comfortable reporting

Secure Coding Practices

Input validation

  • Validate all user inputs including URLs and file uploads
  • Use whitelisting over blacklisting

Authentication and authorisation

  • Ensure only authorised users can access specific resources
  • Use Role Based Access Control (RBAC) to restrict user permissions
  • Avoid hardcoding credentials, use environment variables or secret managers

Data encryption

  • Protect data at rest or in transit
  • Use HTTPS for secure communications
  • Encrypt sensitive data using industry standards like RSA

Dependency management

  • Vulnerable third-party libraries can be exploited
  • Regularly update and patch dependencies
  • Use tools like 'OWASP Dependency Check' to identify vulnerabilities
  • Remove unused or unnecessary dependencies

Secure Development Lifecycle (SDLC)

  • Conduct regular security code reviews
  • Perform static and dynamic application security testing
  • Integrate security checks in CI/CD pipeline

How Security Champions Can Make a Difference

  • Promoting best practices
  • Acting as first responders
  • Fostering a security-first culture
  • Bridging the gap between the security department and the organisation
